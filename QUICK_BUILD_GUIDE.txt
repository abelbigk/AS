 COMMON ISSUES & FIXES
═══════════════════════════════════════════════════════════════════════════════

Issue: "Gradle Sync Failed"
──────────────────────────
Solution:
1. File → Invalidate Caches... → Invalidate and Restart
2. Wait 5 minutes
3. Click "Sync Now" again

Issue: "Build taking 30+ minutes"
─────────────────────────────────
This is normal for first build. Wait.
Subsequent builds: 3-8 minutes
CPU-dependent. Close other apps.

Issue: "Could not resolve dependency"
──────────────────────────────────────
Solution:
1. Delete: c:\mycode3\app\android\.gradle\
2. Delete: c:\mycode3\app\node_modules\
3. Run: npm install
4. Click "Sync Now" in Android Studio

Issue: "C++ Compilation Error"
──────────────────────────────
Already fixed! Hermes is configured.
If still happens:
1. Ensure NDK 26.1.10909125 installed
2. Delete .gradle folder
3. Retry build

 AUTHENTICATION FLOW
═══════════════════════════════════════════════════════════════════════════════

First Launch:
1. App starts → No token in AsyncStorage
2. RootNavigator shows LoginScreen
3. User enters email + password
4. Backend validates (https://as-wryo.onrender.com/api/trpc/auth.login)

Subsequent Launches:
1. App starts
2. authStore.restoreToken() reads from AsyncStorage
3. Token found → Skip login
4. Collections tab shows immediately
