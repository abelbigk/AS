================================================================================
                    DEPLOYED BACKEND - HOW IT WORKS
================================================================================

WHAT YOU WANT:
Your app connects to a live URL (not localhost)
Like: https://my-backend-api.com/auth/me

HOW IT WORKS:

================================================================================
                         CURRENT SETUP
================================================================================

Local Development:

User PC:
  ├─ Backend (npm run dev on port 3000)
  │  └─ http://localhost:3000
  │
  └─ Emulator/Phone
     └─ App connects to localhost:3000

API Call Flow:
  App → http://localhost:3000/auth/me
  Backend Response → Returns to app

Problem: Only works on your PC

================================================================================
                       DEPLOYED SETUP
================================================================================

After Deployment:

GitHub Repository
  └─ Your backend code
     └─ Automatically deploys when you push

Deployment Platform (Railway/Vercel/Heroku)
  └─ Runs your backend 24/7
  └─ Gives you public URL
  └─ Example: https://my-backend.up.railway.app

API Calls Now:
  App (anywhere) → HTTPS://my-backend.up.railway.app/auth/me
  Backend Response → Returns to app

Benefit: Works anywhere, anytime, for anyone

================================================================================
                    STEP-BY-STEP EXPLAINED
================================================================================

STEP 1: PUSH TO GITHUB

Your Local PC:
  ├─ Backend code
  │  ├─ index.js
  │  ├─ package.json
  │  ├─ routes/
  │  └─ models/
  │
  └─ Git push to GitHub
     └─ Your code now on GitHub

GitHub Repository:
  └─ Stores your backend code
     └─ Can be public or private

STEP 2: DEPLOY TO PLATFORM

Platform (Railway/Vercel/Heroku):
  1. Connects to your GitHub repo
  2. Detects it's Node.js
  3. Installs dependencies (npm install)
  4. Starts your server (node index.js or npm run dev)
  5. Assigns public URL
  6. Server runs 24/7

STEP 3: GET PUBLIC URL

Platform gives you something like:
  https://my-backend-project-123.up.railway.app

This URL:
  ✅ Is public (accessible from anywhere)
  ✅ Connects to your running backend
  ✅ Stays up 24/7
  ✅ Auto-updates when you push new code (usually)

STEP 4: UPDATE APP CONFIG

App configuration file (.env):
  BEFORE: EXPO_PUBLIC_API_URL=http://localhost:3000
  AFTER:  EXPO_PUBLIC_API_URL=https://my-backend-project-123.up.railway.app

STEP 5: REBUILD APP

App now uses deployed URL instead of localhost
  App loads ↓ API Call ↓ https://deployed-url.com/auth/me

================================================================================
                      FILE CONFIGURATION
================================================================================

APP FILE: c:\mycode3\mobile-rn\.env

Content:
  EXPO_PUBLIC_API_URL=http://localhost:3000

This tells app where backend is:
  ├─ If = http://localhost:3000 → connects to your PC
  ├─ If = https://backend-api.com → connects to deployed server
  └─ If empty → defaults to http://localhost:3000

How it's used (in code):

  File: src/api/client.ts
  
  const API_BASE_URL = process.env.EXPO_PUBLIC_API_URL || 'http://localhost:3000';
  
  Meaning:
    1. Check .env for EXPO_PUBLIC_API_URL
    2. If found: Use that URL
    3. If not found: Use http://localhost:3000 (default)

                    WORKFLOW AFTER DEPLOYMENT

NORMAL DEVELOPMENT:

================================================================================
                      IMPORTANT NOTES
================================================================================

HTTPS REQUIRED:
  Development: http://localhost:3000 (OK)
  Production: https://... (MUST)
  
  Why: Devices don't trust HTTP in production

CORS:
  Backend must allow requests from your app domain
  Add to backend: app.use(cors());

ENVIRONMENT VARIABLES:
  If backend uses .env vars (JWT_SECRET, DATABASE_URL)
  Set them on deployment platform too
  
DATABASE:
  If using local database (SQLite, etc)
  → Won't work on cloud (local file system)
  
  Solution: Use cloud database (MongoDB Atlas, Firebase, etc)

CUSTOM DOMAIN:
  After deployment, can add custom domain
  Example: https://api.yourdomain.com
  Costs: $10-15/year for domain name
